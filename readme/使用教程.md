# Vue 3 前端项目 — 使用教程

---

## 一、项目简介

基于 **Vue 3 + Vite** 的账号交易平台前端，使用 Pinia 状态管理、Vue Router 路由、Axios 网络请求。包含商城、发布、订单、钱包、管理后台等完整功能。

### 启动

```bash
npm install
npm run dev       # http://localhost:5173
npm run build     # 生产构建 → dist/
```

> 需要**同时启动后端**，见 `account-trading-api/docs/使用教程.md`

---

## 二、页面与路由

| 路由 | 页面 | 权限 |
|------|------|------|
| `/` | 重定向到商城 | 公开 |
| `/trade/items` | 账号商城 | 公开 |
| `/trade/items/:id` | 商品详情 + 购买 | 公开 |
| `/trade/login` | 登录/注册（手机号/用户名均可） | 公开 |
| `/trade/my-items` | 我的发布（**交付在这里**） | 需登录 |
| `/trade/publish` | 发布/编辑商品 | 需登录 |
| `/trade/orders` | 我的订单（仅买家） | 需登录 |
| `/trade/orders/:id` | 订单详情（支付/收货/评价/查看账号信息） | 需登录 |
| `/trade/profile` | 个人中心（钱包/充值/交易记录） | 需登录 |
| `/trade/admin/audits` | 商品审核 | 管理员 |
| `/trade/admin/users` | 用户管理 + 一键清除 | 管理员 |

## 三、组件架构

```
App.vue
├── AppHeader.vue          # 顶部栏（登录态/用户信息/超管徽章/退出）
├── AppSidebar.vue         # 侧边栏（菜单 + 管理员分区）
│   ├── 用户菜单
│   └── ⚙️ 管理分区（管理员可见）
└── router-view
    └── 各页面组件
```

通信方式：
- **Props + Emit**：父子组件
- **v-model**：双向绑定（搜索、表单）
- **Pinia Store**：跨页面共享（登录态、角色）
- **路由守卫**：`requiresAdmin` → 非管理员跳转登录页

## 四、核心功能说明

### 4.1 权限体系

| 角色 | 侧边栏 | 功能 |
|------|--------|------|
| ⭐ 超管 | + ⚙️ 管理分区 | 审核商品、管理用户（升降/删除）、一键清除数据、发布免审核 |
| 👑 管理员 | + ⚙️ 管理分区 | 审核商品、管理普通用户 |
| 👤 普通用户 | 仅用户菜单 | 商城/发布/订单/钱包 |

顶部栏超管登录时显示 ⭐ 超级管理员 徽章。

### 4.2 交易流程

```
卖家发布 → 审核上架 → 买家下单 → 支付扣款
                                  ↓
                           卖家「我的发布」交付账号
                                  ↓
                           买家订单详情确认收货（钱转卖家）
                                  ↓
                           买家评价
                                  ↓
                  2分钟未支付 → 自动取消
```

### 4.3 钱包

- 个人中心显示余额，支持模拟充值
- 交易记录：充值/支付/收入
- 下单不扣费，支付时扣买家余额
- 确认收货后转账给卖家

### 4.4 管理员功能

- **商品审核**：待审核列表，通过/拒绝
- **用户管理**：搜索用户、切换角色（超管可升降管理员）、删除用户
- **一键清除**：清除所有商品/订单/评价/交易记录（仅超管，在用户管理页底部）

## 六、配置说明

### Axios

`src/api/tradeHttp.js` — 后端地址 `http://localhost:3001/api`，自动注入 Token，统一错误处理。

### 路由守卫

`src/router/index.js` — 管理员页面 `meta.requiresAdmin`，非管理员自动跳转登录。

### 路径别名

`@` → `src/`，配置在 `vite.config.js`。

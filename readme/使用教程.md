# Vue 3 前端项目 — 使用教程

---

## 一、项目简介

本项目是一个基于 **Vue 3 + Vite** 构建的前端单页应用（SPA），主要演示以下核心技术：

| 技术 | 版本 | 作用 |
|------|------|------|
| Vue 3 | ^3.4.27 | 渐进式 JavaScript 框架 |
| Vite | ^5.2.12 | 下一代前端构建工具，极速热更新 |
| Vue Router | ^4.3.2 | 官方路由管理器 |
| Axios | ^1.7.2 | 基于 Promise 的 HTTP 请求库 |
| Pinia | ^2.1.7 | Vue 官方状态管理库 |

项目包含 **首页、仪表盘、用户管理、关于项目** 四个功能模块，涵盖了组件化设计、Axios 网络请求、路由配置与守卫等核心知识点。

## 二、环境准备

### 2.1 安装 Node.js

1. 访问 Node.js 官网：https://nodejs.org
2. 下载 **LTS（长期支持版）**，推荐 v18 或更高版本
3. 安装时保持默认选项，一路点击"下一步"即可
4. 安装完成后，打开终端（PowerShell 或 CMD），验证安装是否成功：

```bash
node -v
# 输出示例：v18.20.2

npm -v
# 输出示例：10.5.0
```

> **说明**：npm 是 Node.js 自带的包管理工具，安装 Node.js 时会自动安装 npm，无需单独配置。

### 2.2 安装代码编辑器（可选）

推荐使用 **Visual Studio Code**（VS Code），下载地址：https://code.visualstudio.com

VS Code 安装后可安装以下实用插件：

- **Vue - Official**（原 Volar）— Vue 3 官方插件，提供语法高亮和智能提示
- **ESLint** — 代码规范检查

## 三、项目安装与启动

### 3.1 获取项目

将项目文件夹拷贝到本地，或从 Git 仓库克隆。项目根目录结构如下：

```
qianduan/
├── index.html              # 入口 HTML 文件
├── package.json            # 项目配置（依赖、脚本命令）
├── vite.config.js          # Vite 构建配置
├── public/                 # 静态资源（不会被 Vite 处理）
└── src/                    # 源代码目录
    ├── main.js             # 应用入口
    ├── App.vue             # 根组件
    ├── style.css           # 全局样式
    ├── api/                # Axios 网络层
    ├── router/             # 路由配置
    ├── components/         # 公共组件
    └── views/              # 页面视图
```

### 3.2 安装依赖

打开终端，进入项目根目录，执行：

```bash
cd "C:\Users\qq271\Desktop\前端开发\qianduan"
npm install
```

> **说明**：`npm install` 会读取 `package.json` 中的 `dependencies` 和 `devDependencies`，自动下载所有依赖包到 `node_modules/` 目录。首次执行可能需要 1-3 分钟，取决于网络速度。

安装完成后，你会看到类似如下输出：

```
added 62 packages in 1m
```

### 3.3 启动开发服务器

```bash
npm run dev
```

启动成功后，终端会显示：

```
VITE v5.x.x  ready in 393 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

在浏览器中打开 **http://localhost:5173** 即可看到项目界面。

### 3.4 停止开发服务器

在终端按 **Ctrl + C** 即可停止。

如果终端已被关闭，可通过以下命令终止：

```bash
# 查找占用 5173 端口的进程
netstat -ano | findstr :5173

# 终止进程（将 PID 替换为上一步查到的数字）
taskkill /PID <进程ID> /F
```

### 3.5 生产构建

```bash
npm run build
```

构建完成后，会在项目根目录生成 `dist/` 文件夹，包含优化后的 HTML、CSS、JS 文件，可直接部署到任何静态服务器。

## 四、页面功能使用指南

### 4.1 首页（/）

**访问方式**：点击侧边栏「🏠 首页」

**功能说明**：
- 展示项目的技术栈标签（Vue 3、Vite 5、Vue Router 4、Axios、Pinia）
- 展示三大功能模块的特性卡片（Axios 配置、Vue Router 路由、项目工程化）
- 提供快速开始命令参考

### 4.2 仪表盘（/dashboard）

**访问方式**：点击侧边栏「📊 仪表盘」

**功能说明**：
- 展示四张统计卡片：用户总数、活跃用户、待处理任务、系统可用率
- **权限验证演示**：此页面设置了 `meta.requiresAuth = true`，全局导航守卫会检查 Token
- 点击「模拟登录」按钮，会在 localStorage 写入 Token（F12 → Application → Local Storage 可查看）
- 点击「模拟退出」按钮，会清除 Token
- 打开浏览器控制台（F12 → Console），可以看到路由守卫的输出日志

### 4.3 用户管理（/users）★ 核心功能

**访问方式**：点击侧边栏「👥 用户管理」

**功能说明**：

这是项目的核心模块，展示了完整的组件化设计和 Axios 数据请求。

#### （1）查看用户列表
- 页面加载时自动调用 Axios 请求 `https://jsonplaceholder.typicode.com/users` 获取 10 条真实用户数据
- 每条用户显示：头像、姓名、用户名、邮箱、公司、城市、状态
- 状态徽章有三种颜色：绿色（正常）、红色（禁用）、黄色（待审核）

#### （2）搜索过滤
- 在搜索框中输入关键词（支持按姓名、用户名、邮箱搜索）
- 点击状态筛选按钮（全部/正常/禁用/待审核）过滤列表
- 搜索框右侧的 ✕ 按钮可清空搜索词

#### （3）新增用户
- 点击页面右上角「➕ 新增用户」按钮
- 在弹窗中填写姓名、用户名、邮箱、电话、网站、状态
- 表单会自动校验：姓名/用户名/邮箱为必填，邮箱需符合格式
- 点击「确认新增」提交（模拟 POST 请求）

#### （4）编辑用户
- 点击用户行右侧的 ✏️ 按钮
- 弹窗自动填充该用户的现有信息
- 修改后点击「保存修改」提交（模拟 PUT 请求）

#### （5）删除用户
- 点击用户行右侧的 🗑️ 按钮
- 弹出确认对话框，显示要删除的用户名
- 点击「确认删除」执行删除（模拟 DELETE 请求）

#### （6）查看用户详情（动态路由）
- 点击用户行任意位置（或 👁️ 按钮）
- 跳转到 `/users/:id` 动态路由页面
- 展示该用户的完整信息：用户名、邮箱、电话、网站、地址、公司
- 页面底部显示当前路由参数提示

#### （7）查看用户资料（嵌套动态路由）
- 在用户详情页或用户列表中点击「查看资料」
- 跳转到 `/users/:id/profile` 嵌套动态路由页面
- 展示更详细的资料：公司信息、地址、联系方式、网站

### 4.4 关于项目（/about）

**访问方式**：点击侧边栏「ℹ️ 关于项目」

**功能说明**：
- 项目说明与技术栈
- 环境要求（Node.js ≥ 16）
- 四步启动流程详解
- 完整的项目目录结构树

### 4.5 404 页面

**访问方式**：在地址栏输入任意不存在的路径，如 `http://localhost:5173/abc`

**功能说明**：
- 通过通配符路由 `/:pathMatch(.*)*` 匹配所有未定义的路径
- 显示当前访问的无效路径
- 提供「返回首页」按钮

## 七、常用开发操作

### 7.1 添加新的页面路由

1. 在 `src/views/` 下创建新的 `.vue` 文件
2. 在 `src/router/index.js` 的 `routes` 数组中添加路由配置：

```javascript
{
  path: '/new-page',
  name: 'NewPage',
  component: () => import('@/views/NewPage.vue'),  // 懒加载
  meta: { title: '新页面' }
}
```

3. 在 `src/components/AppSidebar.vue` 的 `navItems` 数组中添加导航项：

```javascript
{ path: '/new-page', icon: '🆕', label: '新页面' }
```

### 7.2 添加新的 API 接口

在 `src/api/` 下创建新文件或在现有文件中添加：

```javascript
// src/api/product.js
import http from './index'

export function getProducts() {
  return http.get('/products')
}

export function getProductById(id) {
  return http.get(`/products/${id}`)
}
```

### 7.3 修改 Axios 基础配置

编辑 `src/api/index.js`：

```javascript
const http = axios.create({
  baseURL: 'http://your-api-server.com/api',  // 修改为你的后端地址
  timeout: 15000,                               // 修改超时时间
})
```

### 7.4 修改开发服务器端口

编辑 `vite.config.js`：

```javascript
server: {
  port: 3000,    // 修改端口号
  host: true,
  open: true
}
```

## 八、常见问题

### Q1：npm install 报错怎么办？

```bash
# 清除缓存后重试
npm cache clean --force
npm install
```

### Q2：端口 5173 被占用怎么办？

```bash
# 查找并终止占用进程
netstat -ano | findstr :5173
taskkill /PID <进程ID> /F
```

或者修改 `vite.config.js` 中的 `server.port` 为其他端口。

### Q3：页面白屏怎么办？

1. 打开 F12 控制台查看是否有红色错误信息
2. 确认 `npm run dev` 正在运行
3. 确认浏览器访问的是 `http://localhost:5173`

### Q4：如何部署到服务器？

```bash
# 1. 构建生产版本
npm run build

# 2. 将 dist/ 文件夹上传到服务器
# 3. 配置 Nginx 或其他 Web 服务器指向 dist/ 目录
```

Nginx 参考配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/dist;
    index index.html;

    # Vue Router History 模式需要此配置
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```
